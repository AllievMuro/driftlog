Catch prompt drift before it breaks your protocol

Driftlog instruments AI agent deployments on Arbitrum with cryptographic fingerprints, behavioral diffs, and an on-chain version registry — so silent model rollovers never silently reprice your liquidations.

The Problem

Model rollovers are silent protocol killers

Your risk management agent passed every audit — on the model version you tested. Then the provider silently rolled to a new snapshot, and now it misprices liquidations at 3 AM.

Unannounced upstream updates

LLM providers push model updates without versioned APIs or changelogs. Your prompt that returned structured JSON last week may return a different schema today.

No behavioral audit trail

On-chain transactions show inputs and outputs — not the agent reasoning that produced them. When something goes wrong, there's no chain of custody to trace.

CI/CD ignores agent drift

Hardhat and Foundry test suites validate Solidity logic. They have no concept of semantic drift in the AI layer sitting above your contracts.

DeFi stakes are asymmetric

A vault-rebalancing agent operating on a drifted model can cause irreversible on-chain damage — liquidation cascades, mispriced options, drained reserves — before any human notices.

How It Works

Three layers of drift protection

Driftlog wraps every AI agent invocation in a cryptographic envelope, replays history against new versions, and gates deployment automatically.

01

Fingerprint Every Invocation

Each agent call is hashed as an EIP-712 typed envelope — prompt hash, model version, tool call schema, output structure — written to Arbitrum calldata via the ArbSys precompile.

EIP-712 · ArbSys · IPFS
02

Behavioral Diff on New Versions

When a new agent version is staged, Driftlog replays curated pinned inputs against both versions and scores semantic divergence. Exceeds threshold? Deployment is blocked automatically.

Diff Runner · Configurable Threshold
03

On-Chain Version Registry

An ERC-7582-compatible registry lets any protocol participant verify which agent spec authorized a given transaction — creating an auditable chain of custody for every AI-driven action.

ERC-7582 · Arbitrum Sepolia
Features

Built for DeFi CI/CD workflows

🔌

Hardhat & Foundry Plugins

Drop-in npm package and forge script wrapper. Drift checks run automatically on every agent pull request inside your existing pipeline — no new infrastructure to maintain. Gate merges on passing behavioral diff scores.

🔐

EIP-712 Fingerprinting

Typed structured data signatures for every invocation. Prompt hash, model version, tool schema, and output structure — all anchored immutably in Arbitrum calldata.

📋

ERC-7582 Registry

Public read interface for third-party verifiers. Any protocol participant can look up which agent spec authorized any transaction by hash.

🔔

Drift Alerts

Webhook and Telegram bot notifications with structured JSON payloads when a threshold breach is detected during staging.

📦

IPFS Prompt Snapshots

Content-addressed prompt storage. Retrieve any historical agent input by its invocation hash — permanent, verifiable, tamper-evident.

📊

Protocol Dashboard

Per-protocol agent version history, drift event log, and invocation fingerprint lookup by transaction hash. Coming Q3 2026.
